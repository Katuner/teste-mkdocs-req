# Histórias de Usuário

!!! info "Unidade 2"
    Esta seção será preenchida na **Unidade 2**.

As histórias de usuário descrevem os requisitos funcionais do ponto de vista do usuário, no formato:

> **Como** [persona], **quero** [ação], **para que** [benefício].

Cada história de usuário será acompanhada de **critérios de aceitação** que definem quando a história está completa, conforme os critérios de **DoR** (Definition of Ready) e **DoD** (Definition of Done).
